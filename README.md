# VacumBook

**VacumBook** — это консольный парсер, который собирает информацию о книгах (название, цена, автор, ссылка) из трех популярных российских интернет-магазинов и сохраняет результаты в локальную SQLite базу данных.

## 📚 Возможности

- 🔍 **Поиск книг:** Введите название книги, и скрипт найдет совпадения.
- 📦 **Мульти-источники:** Парсинг данных с сайтов:
  - [book24.ru](https://book24.ru)
  - [labirint.ru](https://www.labirint.ru)
  - [bookvoed.ru](https://www.bookvoed.ru)
- 💾 **Сохранение данных:** Результаты сохраняются в SQLite базу данных для последующего анализа.
- ⏱ **Защита от блокировок:** Добавлены задержки между запросами к разным сайтам.

## 🛠 Технологии

- **Язык:** Python 3.x
- **Парсинг:** BeautifulSoup4, lxml
- **Запросы:** requests
- **База данных:** SQLite3
- **Анонимизация:** fake-useragent

## 🚀 Установка и запуск

### 1. Клонирование или скачивание
Убедитесь, что у вас установлен Python 3.

### 2. Установка зависимостей
Создайте файл `requirements.txt` (или установите пакеты вручную):

```bash
pip install requests beautifulsoup4 lxml fake-useragent
```

### 3. Подготовка базы данных
Скрипт ожидает наличие папки `database`. Создайте её вручную перед запуском:

```bash
mkdir database
```

### 4. Запуск
Запустите главный файл:

```bash
python main.py
```

### 5. Использование
1. После запуска в консоли появится запрос: `Enter book search:`.
2. Введите название книги (например, `Мастер и Маргарита`).
3. Нажмите Enter.
4. Скрипт последовательно опросит магазины (с задержкой 3 секунды) и сохранит данные в `database/database.db`.

## 💾 Структура базы данных

Таблица `BOOK` содержит следующие поля:

| Поле | Тип | Описание |
|------|-----|----------|
| `TITLE` | TEXT | Название книги |
| `PRICE` | INTEGER | Цена в рублях |
| `LINK` | TEXT | Ссылка на товар |
| `AUTHOR` | TEXT | Автор книги |

## ⚠️ Важные примечания

- **Лимиты запросов:** Между запросами к разным сайтам установлена задержка в 3 секунды (`time.sleep(3)`), чтобы снизить риск блокировки по IP.
- **Структура сайтов:** Парсеры зависят от HTML-структуры сайтов. Если сайты обновят дизайн, скрипт может потребовать обновления селекторов BeautifulSoup.
- **База данных:** При каждом запуске `main.py` создается таблица `BOOK`. Если база уже существует, может возникнуть ошибка дублирования таблицы (требуется доработка кода для проверки существования таблицы через `CREATE TABLE IF NOT EXISTS`).
- **Legal:** Используйте парсер в личных целях. Уважайте `robots.txt` и условия использования сайтов-источников.
